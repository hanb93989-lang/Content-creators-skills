# Live Scene Library

## Use Case

Use this when the user is designing a live room, selecting a live-room scene during project research, or asking what other categories can teach their own live room.

## Case Admission Standard

Only treat a live-room scene as reusable if it has one of these signals:

- Live online audience is at least around 100.
- It is an industry-recognized classic case.
- It creates a clear trust, atmosphere, interaction, or conversion mechanism.
- It can be used as a reference during project research.

## Cross-Category Principle

Do not limit live-room research to the user's own product category. Use categories such as apparel, beauty, jewelry, shoes and bags, home goods, 3C, mother/baby, sports, plants, education, top institutions, top creators, entertainment, and game-like live rooms to extract mechanisms.

## What To Extract

For each live-room reference, extract:

1. Category
2. Scene type
3. Host identity
4. Product presentation method
5. Trust object
6. Interaction rhythm
7. Visual proof
8. Transaction trigger
9. What can migrate to the user's category

## Scene Migration Questions

- Is the trust built by people, environment, product proof, crowd, or price?
- Is the audience staying for information, atmosphere, entertainment, status, scarcity, or problem-solving?
- What is repeated visually enough to become memory?
- What object makes the product credible without over-explaining?
- What can be copied as a mechanism, not as a surface style?

## Output Shape

When coaching:

1. Name the user's current live-room scene problem.
2. Pick the nearest reference family.
3. Extract the transferable mechanism.
4. Give one scene adjustment, one interaction adjustment, and one transaction adjustment.
